## Overview

The `LibraryItem` class represents a unified media entry in Tubify's library system, combining both cloud-based media (from external sources like YouTube) and local file representations. This dual-nature design allows the application to track the complete lifecycle of a media item, from its original online source to its downloaded local copy.

<Info>
Location: `es.perelluent.tubify.dto.LibraryItem`
</Info>

## Fields

<ResponseField name="cloudMedia" type="Media">
  Represents the cloud-based media metadata, including information about the original source URL, file name, and MIME type. May be `null` if the item is local-only.
</ResponseField>

<ResponseField name="localFile" type="DownloadedFile">
  Represents the downloaded file stored on disk, including file path, size, and download date. May be `null` if the item hasn't been downloaded yet (cloud-only).
</ResponseField>

<Accordion title="getStatus()">
```java
public String getStatus() {
    if (cloudMedia != null && localFile != null) {
        return "Downloaded & Local";
    } else if (cloudMedia != null && localFile == null) {
        return "Cloud Only";
    } else if (cloudMedia == null && localFile != null) {
        return "Local Only";
    }
    return "Error";
}
```
Determines the synchronization status:
- **"Downloaded & Local"**: Media exists both in cloud and on disk
- **"Cloud Only"**: Media metadata exists but hasn't been downloaded
- **"Local Only"**: File exists locally without cloud metadata
- **"Error"**: Invalid state (neither cloud nor local data)
</Accordion>

## Usage in LibraryTableModel

The `LibraryItem` class is designed to work seamlessly with `LibraryTableModel`, which displays library items in a JTable. The table model uses `LibraryItem`'s computed properties to populate table columns:

```java
// From LibraryTableModel.getValueAt()
LibraryItem item = items.get(rowIndex);

switch (columnIndex) {
    case 0: return item.getMediaMimeType();  // Type column
    case 1: return item.getStatus();          // Origin column
    case 2: return item.getMediaName();       // File Name column
    default: return null;
}
```

## Dual-Nature Design

The `LibraryItem` class implements a flexible dual-nature pattern:

1. **Cloud Media Tracking**: Stores metadata about media from external sources
2. **Local File Management**: Tracks downloaded files on disk
3. **Hybrid State Support**: Can represent items in various states:
   - Not yet downloaded (cloud-only)
   - Downloaded and synchronized
   - Imported local files (local-only)

This design enables features like:
- Tracking download progress
- Comparing cloud and local versions
- Managing offline access
- Syncing library state
