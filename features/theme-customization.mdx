Tubify features a modern, customizable interface powered by FlatLaf (Flat Look and Feel), offering dynamic theme switching between dark and light modes, custom color schemes, and responsive layouts that adapt to any screen size.

## FlatLaf Integration

Tubify uses [FlatLaf](https://www.formdev.com/flatlaf/) as its Look and Feel library, providing a modern, flat design aesthetic that surpasses traditional Swing appearance.

### FlatLaf Setup

FlatLaf is initialized in the application's main method:

```java
// MainWindow.java:338-355
public static void main(String args[]) {
    try {
        // Set up Montserrat font
        InputStream is = MainWindow.class.getResourceAsStream("/fonts/Montserrat-Regular.ttf");
        Font montserrat = Font.createFont(Font.TRUETYPE_FONT, is).deriveFont(13f);
        GraphicsEnvironment ge = GraphicsEnvironment.getLocalGraphicsEnvironment();
        ge.registerFont(montserrat);
        
        // Configure FlatLaf to use Montserrat
        UIManager.put("defaultFont", montserrat);
        
        // Apply FlatDarkLaf as default theme
        com.formdev.flatlaf.FlatDarkLaf.setup();
    } catch (Exception e) {
        System.err.println("No se pudo cargar Montserrat, usando fuente por defecto.");
        com.formdev.flatlaf.FlatDarkLaf.setup();
    }
    Locale.setDefault(Locale.ENGLISH);
    java.awt.EventQueue.invokeLater(() -> new MainWindow().setVisible(true));
}
```

<Note>
  FlatDarkLaf is the default theme on startup. Users can toggle to FlatLightLaf using the theme switcher.
</Note>

## Dark and Light Mode Toggle

Tubify provides seamless theme switching without requiring application restart.

### Theme Toggle Implementation

The `changeTheme()` method dynamically switches between dark and light modes:

```java
// MainWindow.java:306-336
public void changeTheme() {
    try {
        // Determine next theme based on current state
        LookAndFeel nextLaf = com.formdev.flatlaf.FlatLaf.isLafDark()
                ? new com.formdev.flatlaf.FlatLightLaf()
                : new com.formdev.flatlaf.FlatDarkLaf();
        
        // Apply new Look and Feel
        UIManager.setLookAndFeel(nextLaf);
        
        // Update main window components
        SwingUtilities.updateComponentTreeUI(this);
        
        // Update all panels
        if (loginPanel != null) {
            SwingUtilities.updateComponentTreeUI(loginPanel);
        }
        if (libraryPanel != null) {
            SwingUtilities.updateComponentTreeUI(libraryPanel);
        }
        if (downloadPanel != null) {
            SwingUtilities.updateComponentTreeUI(downloadPanel);
        }
        if (preferences != null) {
            SwingUtilities.updateComponentTreeUI(preferences);
        }
        if (aboutPanel != null) {
            SwingUtilities.updateComponentTreeUI(aboutPanel);
        }
        
    } catch (Exception ex) {
        ex.printStackTrace();
    }
}
```

<Steps>
  <Step title="Detect current theme">
    Uses `FlatLaf.isLafDark()` to check if the current theme is dark mode.
  </Step>
  
  <Step title="Select opposite theme">
    Creates `FlatLightLaf` if currently dark, or `FlatDarkLaf` if currently light.
  </Step>
  
  <Step title="Apply new Look and Feel">
    Sets the new theme via `UIManager.setLookAndFeel()`.
  </Step>
  
  <Step title="Update all components">
    Recursively updates the entire component tree for all panels to reflect the new theme.
  </Step>
</Steps>

## Responsive Layouts

Tubify uses MigLayout to create flexible, responsive interfaces that adapt to window resizing.

### MigLayout Configuration

Different panels use MigLayout with specific constraints:

<Tabs>
  <Tab title="Download Panel">
    ```java
    // DownloadPanel.java:49
    setLayout(new MigLayout("fill, insets 10", "[grow]", "[]"));
    
    // Card panel
    JPanel pnlCard = new JPanel(new MigLayout("wrap, insets 20, gapy 10", "[grow, fill]"));
    ```
    
    - `fill`: Panel fills available space
    - `insets 10`: 10-pixel padding
    - `wrap`: Components stack vertically
    - `gapy 10`: 10-pixel vertical gaps
  </Tab>
  
  <Tab title="Library Panel">
    ```java
    // LibraryPanel.java:61
    setLayout(new MigLayout("fill, insets 10 0 10 10", "[grow, fill]", "[][grow][]"));
    ```
    
    Three-row layout:
    - Row 1: Header (fixed height)
    - Row 2: Table (grows to fill space)
    - Row 3: Actions (fixed height)
  </Tab>
  
  <Tab title="Login Panel">
    ```java
    // LoginPanel.java:63
    setLayout(new MigLayout("fill, insets 0", "[center]", "[center]"));
    
    // Form panel
    JPanel pnlForm = new JPanel(new MigLayout("wrap, insets 30, gapy 10", "[grow, fill]"));
    ```
    
    Centered layout with the form panel having generous padding.
  </Tab>
  
  <Tab title="Main Window">
    ```java
    // MainWindow.java:142
    pnlMain.setLayout(new MigLayout("fill, insets 10", "[320!, fill]10[grow, fill]", "[grow]"));
    ```
    
    Two-column layout:
    - Left: Download panel (fixed 320px width)
    - Right: Library panel (grows to fill remaining space)
    - Gap: 10 pixels between columns
  </Tab>
</Tabs>

### Dynamic Component Sizing

MigLayout constraints define how components resize:

```java
// Fixed height constraints
pnlCard.add(txtUrl, "h 35!");           // Exactly 35px height
pnlCard.add(cmbResolucion, "h 30!");    // Exactly 30px height
pnlCard.add(btnDownload, "h 45!, gaptop 15"); // 45px height, 15px top gap

// Growing width
pnlHeader.add(txtSearch, "width 250!");  // Fixed 250px width

// Grow to fill
add(scrollPane, "grow, wrap");           // Table grows both directions
```

### Minimum Window Size

The main window enforces minimum dimensions:

```java
// MainWindow.java:82
this.setMinimumSize(new Dimension(1100, 700));
```

This prevents the UI from collapsing into unusable layouts when resized too small.

public static ImageIcon UpscaleIcon(ImageIcon icon, int width, int height) {
    if (icon == null) {
        return null;
    }

    Image newImg = img.getScaledInstance(width, height, Image.SCALE_SMOOTH);
    return new ImageIcon(newImg);

## Theme Persistence

<Warning>
  Currently, theme preference is not persisted between application sessions. The app always starts in dark mode.
</Warning>

To add theme persistence, you could extend the preferences system:

```java
// Example implementation (not in current code)
prefs.put("theme", "light"); // or "dark"

// On startup
String savedTheme = prefs.get("theme", "dark");
if ("light".equals(savedTheme)) {
    UIManager.setLookAndFeel(new com.formdev.flatlaf.FlatLightLaf());
} else {
    UIManager.setLookAndFeel(new com.formdev.flatlaf.FlatDarkLaf());
}
```

## Customization Tips

<Accordion title="Modifying the brand color">
  Search for `#fb3f62` in the source code and replace with your preferred hex color. Remember to also update the lighter selection color `#f39bab`.
</Accordion>

<Accordion title="Changing the default font">
  Replace the Montserrat TTF file in `/fonts/` and update the font name in `MainWindow.main()`. Ensure the new font file is added to the resources.
</Accordion>

<Accordion title="Adjusting component rounding">
  Modify the `arc:` values in FlatLaf style properties. Higher values create more rounded corners.
</Accordion>

<Accordion title="Adding new FlatLaf themes">
  FlatLaf supports custom themes via properties files. See the [FlatLaf documentation](https://www.formdev.com/flatlaf/themes/) for details.
</Accordion>

## Troubleshooting

<Accordion title="Theme doesn't change">
  Ensure you're using a recent version of FlatLaf. Older versions may not support dynamic theme switching.
</Accordion>

<Accordion title="Custom font not loading">
  Check that:
  - The font file exists in `/fonts/`
  - The file path in the code matches the actual location
  - The font file is included in the build/JAR
</Accordion>

<Accordion title="Components not updating after theme change">
  The `changeTheme()` method should call `SwingUtilities.updateComponentTreeUI()` on all panels. If a panel isn't updating, ensure it's included in the update loop.
</Accordion>

<Accordion title="Colors look wrong in light mode">
  The pink accent color (#fb3f62) works well in both themes. If adding new colors, test them in both dark and light modes for adequate contrast.
</Accordion>
