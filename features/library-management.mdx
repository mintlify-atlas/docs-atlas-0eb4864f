The Library Panel provides a comprehensive interface for managing your media collection, combining both local files and cloud-stored media in a single unified view. Built with a sophisticated table-based interface, it offers powerful search, filtering, and file operation capabilities.

## Library Panel Overview

The `LibraryPanel.java` class creates a unified view that merges local downloaded files with cloud-synchronized media, providing seamless access to your entire media collection.

<Note>
  The library automatically refreshes when new media is detected either locally or in the cloud, thanks to the MediaPollingBean event system.
</Note>

## Unified View Architecture

Tubify's library uses a sophisticated merge system to combine local and cloud files into a single cohesive view.

### LibraryItem Structure

Each row in the library table represents a `LibraryItem` that can contain:
- Local file reference (`DownloadedFile`)
- Cloud media reference (`Media`)
- Or both (synchronized files)

```java
// LibraryPanel.java:686-703
private List<LibraryItem> mergeLists(List<Media> cloud, List<DownloadedFile> local) {
    Map<String, LibraryItem> map = new HashMap<>();
    if (cloud != null) {
        for (Media m : cloud) {
            map.put(m.mediaFileName, new LibraryItem(m, null));
        }
    }
    if (local != null) {
        for (DownloadedFile df : local) {
            String name = df.getFileName();
            if (map.containsKey(name)) {
                map.get(name).setLocalFile(df);
            } else {
                map.put(name, new LibraryItem(null, df));
            }
        }
    }
    return new ArrayList<>(map.values());
}
```

### Regex-Based Filtering

The filter method uses `TableRowSorter` with regex patterns:

```java
// LibraryPanel.java:325-339
private void filter(String query) {
    TableRowSorter<LibraryTableModel> sorter = new TableRowSorter<>(libraryModel);
    tblLibrary.setRowSorter(sorter);
    if (query.length() == 0) {
        sorter.setRowFilter(null);
    } else {
        try {
            String escapedQuery = Pattern.quote(query);
            sorter.setRowFilter(RowFilter.regexFilter("(?i)" + escapedQuery));
        } catch (java.util.regex.PatternSyntaxException e) {
            System.err.println("Error en el patrón de búsqueda: " + e.getMessage());
        }
    }
}
```

<Accordion title="How the filtering works">
  1. **Pattern Escaping**: User input is escaped using `Pattern.quote()` to prevent regex syntax errors
  2. **Case-Insensitive**: The `(?i)` flag enables case-insensitive matching
  3. **Real-Time**: Filtering occurs on every keystroke via the KeyListener
  4. **Column-Wide**: The filter searches across all columns in the table
</Accordion>

    ```java
    private void playCloudMedia(Media media) {
        SwingWorker<File, Void> worker = new SwingWorker<>() {
            @Override
            protected File doInBackground() throws Exception {
                // Creates the temporary file
                String ext = media.mediaFileName.contains(".")
                        ? media.mediaFileName.substring(media.mediaFileName.lastIndexOf(".")) : ".tmp";
                File tempFile = File.createTempFile("tubify_stream_", ext);
                tempFile.deleteOnExit();
                
                main.getMediaPollingBean().download(media.id, tempFile);
                
                return tempFile;
            }
            
            @Override
            protected void done() {
                try {
                    File tempFile = get();
                    Desktop.getDesktop().open(tempFile);
                } catch (Exception ex) {
                    JOptionPane.showMessageDialog(LibraryPanel.this,
                            "Error streaming file: " + ex.getMessage());
                }
            }
        };
        
        this.setCursor(java.awt.Cursor.getPredefinedCursor(java.awt.Cursor.WAIT_CURSOR));
        worker.execute();
    }
    ```
    
    <Note>
      The cursor changes to a wait cursor during cloud file download to provide visual feedback.
    </Note>
  </Tab>
</Tabs>

### Upload Files

Upload local files to the cloud storage:

```java
// LibraryPanel.java:416-475
private void btnUploadActionPerformed(java.awt.event.ActionEvent evt) {
    int viewRow = tblLibrary.getSelectedRow();
    if (viewRow == -1) {
        JOptionPane.showMessageDialog(this, "Please select a local file to upload.");
        return;
    }
    
    int modelRow = tblLibrary.convertRowIndexToModel(viewRow);
    LibraryItem item = libraryModel.getItemAt(modelRow);
    
    // Validation: must have local file
    if (item.getLocalFile() == null) {
        JOptionPane.showMessageDialog(this,
                "This file is only in the Cloud. You cannot upload it again.",
                "Upload Error", JOptionPane.WARNING_MESSAGE);
        return;
    }
    
    // Validation: not already in cloud
    if (item.getCloudMedia() != null) {
        javax.swing.JOptionPane.showMessageDialog(this,
                "This file is already synchronized with the Cloud.",
                "Info", javax.swing.JOptionPane.INFORMATION_MESSAGE);
        return;
    }
    
    SwingWorker<Void, Void> worker = new SwingWorker<>() {
        @Override
        protected Void doInBackground() throws Exception {
            File fileToUpload = new File(item.getLocalFile().getFilePath());
            
            if (!fileToUpload.exists()) {
                throw new FileNotFoundException("File not found: " + fileToUpload.getAbsolutePath());
            }
            
            main.getMediaPollingBean().uploadFileMultipart(fileToUpload, "");
            return null;
        }
        
        @Override
        protected void done() {
            try {
                get();
                JOptionPane.showMessageDialog(LibraryPanel.this,
                        "Upload Successful!",
                        "Success", JOptionPane.INFORMATION_MESSAGE);
                loadMedia();
            } catch (Exception e) {
                JOptionPane.showMessageDialog(LibraryPanel.this,
                        "Upload Failed: " + e.getMessage(),
                        "Error", JOptionPane.ERROR_MESSAGE);
            }
        }
    };
    worker.execute();
}
```

<Warning>
  Upload only works for files that exist locally and are not already synchronized with the cloud.
</Warning>

### Delete Files

Delete local files from disk with confirmation:

```java
// LibraryPanel.java:482-517
private void btnDeleteActionPerformed(java.awt.event.ActionEvent evt) {
    int selectedRow = tblLibrary.getSelectedRow();
    if (selectedRow == -1) {
        JOptionPane.showMessageDialog(LibraryPanel.this, "Please. First select a File to delete.");
        return;
    }
    
    int modelRow = tblLibrary.convertRowIndexToModel(selectedRow);
    LibraryItem item = libraryModel.getItemAt(modelRow);
    
    if (item != null && item.getLocalFile() != null) {
        int confirm = JOptionPane.showConfirmDialog(
                LibraryPanel.this,
                "Are you sure you want to delete '" + item.getMediaName() + "' from your computer?",
                "Confirm Delete",
                JOptionPane.YES_NO_OPTION,
                JOptionPane.WARNING_MESSAGE
        );
        
        if (confirm == JOptionPane.YES_OPTION) {
            boolean deleted = item.getLocalFile().deleteFromDisk();
            
            if (deleted) {
                JOptionPane.showMessageDialog(LibraryPanel.this, "File deleted from disk.");
                loadMedia();
            } else {
                JOptionPane.showMessageDialog(LibraryPanel.this,
                        "The file could not be deleted. It might be in use by another program.",
                        "Error", JOptionPane.ERROR_MESSAGE);
            }
        }
    } else {
        JOptionPane.showMessageDialog(LibraryPanel.this,
                "This file is only in the Cloud. Local deletion is not possible.");
    }
}
```

<Note>
  Delete operations only affect local files. Cloud-only files cannot be deleted from the library panel.
</Note>

## Auto-Scroll and Selection

One of Tubify's standout features is automatic selection and scrolling to newly downloaded files.

### Implementation

After a download completes, the library automatically:

1. Refreshes the media list
2. Searches for the newly downloaded file
3. Selects the matching row
4. Scrolls the table to make it visible

```java
// LibraryPanel.java:625-659
if (fileToSelect != null && !fileToSelect.isEmpty()) {
    String searchName = fileToSelect.contains(".")
            ? fileToSelect.substring(0, fileToSelect.lastIndexOf("."))
            : fileToSelect;
    
    // 100ms Timer to ensure the JTable has finished rendering
    javax.swing.Timer selectionTimer = new javax.swing.Timer(200, e -> {
        String cleanTarget = fileToSelect
                .replaceAll("\\.f\\d+\\.[a-z0-9]+$", "") // Remove temporal yt-dlp
                .replaceAll("\\.[^.]+$", "") // Remove extension
                .replaceAll("[^a-zA-Z0-9]", "") // Remove non-alphanumeric
                .toLowerCase();
        
        for (int i = 0; i < tblLibrary.getRowCount(); i++) {
            Object value = tblLibrary.getValueAt(i, 2);
            if (value != null) {
                String nameInTable = value.toString()
                        .replaceAll("\\.[^.]+$", "")
                        .replaceAll("[^a-zA-Z0-9]", "")
                        .toLowerCase();
                
                // Proximity comparison
                if (!cleanTarget.isEmpty() && 
                    (nameInTable.contains(cleanTarget) || cleanTarget.contains(nameInTable))) {
                    final int row = i;
                    tblLibrary.setRowSelectionInterval(row, row);
                    Rectangle rect = tblLibrary.getCellRect(row, 0, true);
                    tblLibrary.scrollRectToVisible(rect);
                    tblLibrary.requestFocusInWindow();
                    return;
                }
            }
        }
    });
    selectionTimer.setRepeats(false);
    selectionTimer.start();
}
```

<Accordion title="Why use a Timer?">
  The 200ms delay ensures the JTable has finished rendering all rows before attempting to select and scroll. This prevents race conditions where the selection occurs before the table is fully populated.
</Accordion>

### Fuzzy Matching

The auto-selection uses fuzzy matching to handle:
- Temporary yt-dlp file extensions (`.f123.mp4`)
- Different file extensions
- Special characters in filenames
- Case variations

All media loading operations run in background threads to keep the UI responsive:

```java
            final List<LibraryItem> mergedList = mergeLists(cloudList, localList);
            
            // Update UI on EDT
            SwingUtilities.invokeLater(() -> {

                // Auto-select logic here
```

<Note>

## Troubleshooting

<Accordion title="Library is empty">
  Check that:
  - The library path is correctly set in Preferences
  - You have downloaded at least one file
  - You are logged in to see cloud files
</Accordion>

<Accordion title="Cannot play file">
  Local files: Ensure the file still exists at the original location.
  
  Cloud files: Check your internet connection and verify you're logged in.
</Accordion>

<Accordion title="Upload fails">
  Verify that:
  - The file exists locally
  - The file is not already in the cloud
  - You have an active internet connection
  - Your JWT token is still valid
</Accordion>

<Accordion title="Search not working">
  The search uses regex patterns. Special characters are automatically escaped, but try simplifying your search query.
</Accordion>
